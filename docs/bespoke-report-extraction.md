# Bespoke Report Extraction

**Date:** 2026-03-09
**Status:** Design

## Problem

The 27-phase pipeline produces comprehensive HTML reports — 10–25 sections each, covering convergence diagnostics, full legislator tables, correlation plots, and interactive visualizations. These are the right output for analysis. They are the wrong output for presentations, stakeholder briefings, or answering a specific research question.

When a journalist asks "show me where Huelskamp sits relative to the rest of the 79th Senate," the answer is section 15 of the 06_IRT_2D report. But sharing that means either sending the entire 2MB report or manually extracting HTML — tedious, error-prone, and not reproducible.

## Solution

A CLI tool (`tallgrass-extract`) that extracts one or more sections from existing pipeline HTML reports into standalone, self-contained HTML files. The tool operates on finished HTML (no re-computation), outputs to a separate `results/bespoke/` directory, and preserves all interactivity (Plotly hover, DataTables sorting).

## Design Principles

1. **Parse, don't re-render.** The HTML reports are machine-generated with predictable structure. Extracting sections from the finished HTML is fast, reliable, and doesn't require loading data or re-running models.

2. **Separate output directory.** Bespoke extractions go to `results/bespoke/<date>/`, never touching the pipeline's `results/kansas/` tree. A `latest` symlink follows the existing convention.

3. **Sections are the unit of extraction.** Every report section has a unique `id` attribute and a sequential number. Both are valid selectors.

4. **Preserve interactivity.** Plotly charts, ITables, and DataTables all work in the extracted output. CDN script tags are deduplicated and hoisted into `<head>`.

## CLI Interface

```bash
# List all sections in a report
just extract results/kansas/79th_2001-2002/latest/06_irt_2d/06_irt_2d_report.html --list

# Extract by section number
just extract results/kansas/79th_2001-2002/latest/06_irt_2d/06_irt_2d_report.html \
    --section 15

# Extract by section ID
just extract results/kansas/79th_2001-2002/latest/06_irt_2d/06_irt_2d_report.html \
    --section interactive-2d-scatter-senate

# Extract multiple sections
just extract results/kansas/79th_2001-2002/latest/06_irt_2d/06_irt_2d_report.html \
    --section 15 --section 16

# Custom title and output path
just extract results/kansas/79th_2001-2002/latest/06_irt_2d/06_irt_2d_report.html \
    --section 15 \
    --title "79th Senate: 2D Ideal Points" \
    --output senate-huelskamp.html
```

### Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `report` | Yes | Path to source HTML report |
| `--section` | No* | Section ID or number (repeatable). Required unless `--list`. |
| `--list` | No | Print all sections (number, ID, title) and exit |
| `--title` | No | Custom title for extracted report (default: "Extracted: {original title}") |
| `--output` | No | Explicit output path (default: `results/bespoke/<date>/<slug>.html`) |

## HTML Parsing Strategy

Pipeline reports have a predictable structure generated by `ReportBuilder` in `analysis/report.py`:

```html
<section class="report-section" id="SECTION_ID">
    <h2><span class="section-number">N.</span> Section Title</h2>
    ... content (tables, figures, Plotly divs, text) ...
</section>
```

A regex captures all four components (ID, number, title, content) in a single pass:

```python
SECTION_RE = re.compile(
    r'<section class="report-section" id="([^"]+)">\s*'
    r'<h2><span class="section-number">(\d+)\.</span>\s*(.*?)</h2>'
    r'(.*?)</section>',
    re.DOTALL,
)
```

This is safe because the HTML is machine-generated by our own template — not arbitrary user HTML. A comment in the report template documents this dependency.

### Section matching

The `--section` argument supports two modes:

1. **By number:** `--section 15` matches `<span class="section-number">15.</span>`
2. **By ID:** `--section interactive-2d-scatter-senate` matches `id="interactive-2d-scatter-senate"`

Detection is automatic: if the argument is a pure integer, match by number; otherwise match by ID.

## Dependency Handling

### Plotly

Each `InteractiveSection` includes its own Plotly CDN `<script>` tag:

```html
<script src="https://cdn.plot.ly/plotly-2.35.2.min.js"></script>
```

When extracting multiple interactive sections, these would be duplicated. The extractor:

1. Scans all extracted content for Plotly CDN script tags
2. Removes duplicates from section bodies
3. Injects a single copy into `<head>`

### ITables / DataTables

Same pattern — interactive tables embed their own JS/CSS inline. The extractor deduplicates library loads while preserving per-table initialization scripts.

### Base64 images

Already self-contained in `<img src="data:image/png;base64,...">`. No special handling needed.

### CSS

Reuses `REPORT_CSS` from `analysis/report.py` directly — same styling as the full report.

## Output Structure

```
results/
  bespoke/
    2026-03-09/
      79th-06-irt-2d-interactive-2d-scatter-senate.html
      79th-06-irt-2d-sections-15-16.html
    latest -> 2026-03-09/
```

Default slug generation: `{biennium}-{phase}-{section-ids}`. With `--output`, the file is written to the specified path under `results/bespoke/<date>/`.

## Rendered Output

The extracted HTML is a minimal, self-contained page:

- **Header:** Custom title (or "Extracted: {original}"), provenance line ("From 06_IRT_2D Report, 79th_2001-2002, extracted 2026-03-09")
- **TOC:** Only if multiple sections extracted
- **Sections:** Extracted content with original numbering preserved
- **Footer:** Generation timestamp
- **Styling:** Same `REPORT_CSS` as pipeline reports — visually identical

## Example: The Triggering Use Case

The 79th Kansas Senate (2001–02) has a genuinely multidimensional legislature — essentially three parties. The 2D IRT interactive scatter (section 15 of the 06_irt_2d report) shows this clearly: Huelskamp and the insurgent Republicans sit in the lower-right, establishment Republicans in the upper-right, Democrats on the left.

```bash
just extract results/kansas/79th_2001-2002/latest/06_irt_2d/06_irt_2d_report.html \
    --section interactive-2d-scatter-senate \
    --title "79th Kansas Senate: Three Factions"
```

This produces a single-page HTML file with the interactive Plotly scatter — hover to see legislator names, party, and both dimension scores. Suitable for embedding in a presentation or sharing with a reporter.

## Not In Scope

- **Re-rendering from source data.** This tool extracts from finished HTML. If the source report doesn't exist, run the pipeline first.
- **Cross-report extraction.** Each extraction pulls from a single source report. Combining sections from different reports (e.g., IRT + UMAP) would require multiple extractions or a future enhancement.
- **PDF export.** HTML is the output format. Browser print-to-PDF works for static sections; Plotly charts require a screenshot.
